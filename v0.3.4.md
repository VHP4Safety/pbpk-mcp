# PBPK MCP v0.3.4

## Release Summary

- `PBPK MCP v0.3.4: stronger OECD evidence traceability`

## Overview

PBPK MCP `v0.3.4` strengthens the OECD-oriented dossier layer without changing the public workflow shape introduced in `v0.3.x`.

This release adds:

- richer parameter-table provenance coverage through generic companion bundles
- structured peer-review, prior-use, and revision-history normalization
- stronger `modelPerformance` traceability through dataset records and acceptance criteria
- a descriptive `oecdCoverage` mapping to OECD PBK Guidance Tables 3.1 and 3.2

It does **not**:

- change qualification scoring semantics
- move BER calculation or decision logic into PBPK MCP
- publish local/example model assets

## Added

- generic parameter-table companion bundle support for `.pkml` and MCP-ready `.R` models
- starter documentation and template for parameter-table bundles
- additive `oecdCoverage` reporting in the OECD dossier export

## Changed

- normalized `peerReview` metadata into structured review records, prior regulatory use, revision status, and revision history coverage
- normalized `modelPerformance` metadata into structured dataset records and acceptance-criterion coverage
- strengthened boundary/support metadata on PBPK-side NGRA handoff objects for downstream orchestrators

## Why It Matters

PBPK MCP is now a more defensible qualification and reporting substrate for NGRA-style workflows. The public MCP surface is unchanged, but the evidence model is clearer, more traceable, and closer to an OECD-style reporting structure.

That matters because it improves:

- dossier completeness for parameter provenance
- review traceability for prior use and revision history
- predictivity traceability without overstating evidence strength
- downstream orchestration by keeping decision ownership outside the PBPK layer
