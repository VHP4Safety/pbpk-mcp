# PBPK MCP v0.3.3

## Release Summary

- `PBPK MCP v0.3.3: NGRA-ready PBPK objects and external PBPK normalization`

## Overview

PBPK MCP `v0.3.3` extends the `v0.3.x` qualification-oriented workflow with a small PBPK-side NGRA object layer and a normalization-only import path for external PBPK outputs.

This release adds:

- typed PBPK-side NGRA objects in validation and dossier export
- a new `ingest_external_pbpk_bundle` tool for external PBPK result normalization
- a stricter BER handoff boundary so PBPK MCP only marks a BER bundle as ready when a real internal exposure metric and external `podRef` are both present

It does **not**:

- execute commercial PBPK engines such as GastroPlus or Simcyp
- parse proprietary project files
- calculate BER
- make decision recommendations

## Added

- `ingest_external_pbpk_bundle`
- additive `ngraObjects` in:
  - `validate_simulation_request`
  - `export_oecd_report`
- typed PBPK-side objects:
  - `assessmentContext`
  - `pbpkQualificationSummary`
  - `uncertaintySummary`
  - `internalExposureEstimate`
  - dossier-only `berInputBundle`

## Why It Matters

PBPK MCP is now better aligned with a Tier 1 NGRA role: it can emit internal-exposure and qualification outputs in a stable typed form, while keeping BER calculation and decision logic outside the PBPK execution layer.

That matters because it preserves clean boundaries:

- PBPK MCP owns PBPK execution, qualification, verification, and PBPK-side evidence normalization
- downstream orchestrators can consume those objects for BER calculation and broader decision workflows
- external PBPK outputs can be normalized into the same object family without pretending PBPK MCP ran or qualified the upstream engine
