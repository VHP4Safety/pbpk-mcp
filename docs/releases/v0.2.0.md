# PBPK MCP v0.2.0

Suggested GitHub release title:

- `PBPK MCP v0.2.0: dual-backend discovery and OECD-oriented validation`

Suggested short release summary:

- PBPK MCP now supports both OSPSuite `.pkml` and MCP-ready `rxode2` `.R` models through one user-facing workflow, with automatic model discovery and explicit OECD-oriented qualification metadata.

## Highlights

- `rxode2` is now a first-class execution backend for custom and population PBPK models authored directly in R
- supported model files are discoverable from disk without preloading through `discover_models` and `/mcp/resources/models`
- `validate_simulation_request` now provides explicit context-of-use and applicability checks before execution
- OSPSuite `.pkml` models can carry sidecar-based scientific profile metadata
- enhanced MCP responses now expose a stable `pbpk-mcp.v1` contract for client chaining

## Included In This Release

### Dual-backend PBPK execution

- `ospsuite` for PK-Sim and MoBi simulation-transfer files (`.pkml`)
- `rxode2` for MCP-ready custom and population-oriented R models (`.R`)

### Discovery and workflow improvements

- automatic filesystem-backed model discovery
- explicit separation between discoverable model files and loaded sessions
- enriched discovery entries after load

### OECD-oriented qualification improvements

- declared `profile` metadata for context of use, applicability domain, uncertainty, verification, and review
- preflight `validate_simulation_request` with `oecdChecklist` and `oecdChecklistScore`
- clearer distinction between runnable models and scientifically qualified models

## Limitations To Call Out Publicly

- Berkeley Madonna `.mmd` is not directly executable
- only `.pkml` and MCP-ready `.R` are direct runtime formats
- runtime support does not imply regulatory qualification
- not all example models ship a complete external validation dossier
- `rxode2` worker image builds are heavy and should be prebuilt

## Suggested Release Notes Body

```markdown
PBPK MCP `v0.2.0` introduces a dual-backend architecture under one user-facing MCP workflow.

This release adds direct support for:

- OSPSuite execution of PK-Sim / MoBi `.pkml`
- `rxode2` execution of MCP-ready `.R` PBPK models

It also adds:

- automatic model discovery from disk through `discover_models` and `/mcp/resources/models`
- OECD-oriented scientific `profile` metadata
- preflight `validate_simulation_request` assessments with `oecdChecklist` coverage
- improved async result chaining and response-shape stability through `pbpk-mcp.v1`

Important limitations remain explicit:

- Berkeley Madonna `.mmd` is a conversion source, not a runtime backend
- `research-use` and `illustrative-example` models are not automatically regulatory-ready
- runtime guardrails are not a substitute for external scientific qualification
```
