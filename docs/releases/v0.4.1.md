# PBPK MCP v0.4.1

## Release Summary

- `PBPK MCP v0.4.1: packaged-default local runtime convergence`

## Overview

PBPK MCP `v0.4.1` is a focused runtime and release-convergence patch on top of `v0.4.0`.

This release makes the documented local stack packaged by default, while preserving an explicit maintainer-only source-overlay path for workspace iteration.

It does **not** change:

- the published `pbpk-mcp.v1` contract
- the public PBPK workflow or tool catalog
- the PBPK/NGRA boundary
- BER calculation or assessment decision ownership

## Changed

- `docker-compose.celery.yml` now represents the packaged local runtime directly
- `docker-compose.overlay.yml` plus `scripts/deploy_source_overlay_stack.sh` now define the explicit source-overlay maintainer profile
- the `rxode2` worker image installs the current `mcp-bridge` package into site-packages during image build
- `scripts/runtime_src_overlay.pth` only activates `/app/src` when `PBPK_ENABLE_SRC_OVERLAY=true`
- deployment and worker-image docs now describe the packaged-default local runtime and the separate maintainer overlay path more explicitly

## Why It Matters

This closes the most obvious remaining release/packaging inconsistency after `v0.4.0`.

The public repo now says, and the local runtime now actually does, the same thing:

- packaged runtime by default
- explicit overlay only when a maintainer opts into it

That makes the project easier to trust operationally and reduces confusion around what is part of the shipped package versus what is only part of a local workspace override.

```bash
curl -s http://127.0.0.1:8000/health | jq .
curl -s http://127.0.0.1:8000/mcp/list_tools | jq '.tools | length'
curl -s http://127.0.0.1:8000/mcp/resources/contract-manifest | jq .
```

Those checks confirmed the live packaged stack reported `version: 0.4.1` from `/health` and continued to serve the expected public contract resources.
