# PBPK MCP v0.4.3

## Release Summary

- `PBPK MCP v0.4.3: trust-surface hardening, benchmark-guided dossier review, and release-evidence tightening`

## Overview

PBPK MCP `v0.4.3` is the first release in this line that fully reflects the recent safety, trust-architecture, and benchmark-guided documentation work.

This release is not just a runtime hotfix. It materially strengthens how the MCP:

- separates runnable from scientifically supported
- surfaces missing evidence and missing documentation
- keeps thin summaries from overclaiming
- preserves sign-off and audit evidence
- compares internal research-use examples against an external public-code benchmark bar without promoting them to regulatory-ready status

## Added

- a regulatory gold-set benchmark dossier under `benchmarks/regulatory_goldset/`
- advisory `curationSummary.regulatoryBenchmarkReadiness` output for manifest validation and trust-bearing summaries
- a model-onboarding checklist for new PBPK chemicals under `docs/pbpk_model_onboarding_checklist.md`
- stronger benchmark-linked audit content in the internal live audit flow

## Changed

- retired the bundled analyst UI from the public/runtime surface; trust-bearing review remains on MCP and REST payloads
- tightened anti-misread, caution, summary-transport, export-block, and sign-off surfaces across validation and OECD report export
- made release/readiness integrity checks stricter by requiring the live runtime to match the published contract artifacts after redeploy

## What This Release Does Not Claim

`v0.4.3` does **not** make research-use examples automatically regulatory-grade.

It makes PBPK MCP better at showing:

- what is explicit
- what is missing
- what remains bounded
- what stronger downstream claim is still blocked

Formal regulatory-grade use still depends on the underlying model package, evidence, uncertainty treatment, and external review.


Those checks confirmed:

- release metadata alignment at `0.4.3`
- packaged `sdist`/wheel integrity
- current benchmark dossier integrity
- current contract-manifest integrity
- a live runtime whose contract-manifest hash matches the published artifact

## Why It Matters

PBPK MCP is now substantially safer and more useful than the earlier public release line because it no longer acts only as an execution wrapper.

It now acts as a trust and legibility layer that can:

- execute PBPK models
- expose missing documentation and missing evidence
- preserve review and audit traces
- discourage overclaiming before a model is truly ready for stronger use
